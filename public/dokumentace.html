Kapitola 1

Úvod

1.1 Motivace a cíle projektu

Cylindrické projekce — velkoformátové scény obtočené kolem diváka — se stávají stále rozšířenějším médiem pro prezentace, výstavy, planetária a imerzivní instalace. Příprava obsahu pro tyto scény je však složitá: běžné grafické editory pracují s plochou plochou a neumožňují designérovi přirozeně vizualizovat, jak bude výsledek vypadat po rozvinutí do cylindru.

ARena je editorský nástroj navržený přímo pro tento use-case. Umožňuje designérovi pracovat na rozvinutém povrchu cylindru v 2D pohledu (přesná kompozice, vrstvení, pozicování) a zároveň kdykoliv přepnout do 3D náhledu pro kontrolu výsledku z pohledu diváka uvnitř cylindru.

Hlavní cíle projektu:

  • Poskytnout intuitivní 2D canvas editor pro tvorbu obsahu 360° cylindrických scén
  • Vizualizovat výsledek v reálném čase jako 3D cylindr
  • Vypočítat a zobrazit bezpečnou zónu — oblast viditelnou pro diváka dané výšky
  • Umožnit tvorbu interaktivních hotspotů pro navigaci mezi scénami
  • Pracovat plně v prohlížeči, bez nutnosti serveru (local-first přístup)
  • Exportovat scény do formátů vhodných pro nasazení (ZIP, JSON, PNG)

1.2 Existující řešení a jejich omezení

Pro tvorbu obsahu pro cylindrické projekce neexistuje žádný standardní specializovaný nástroj. Designéři jsou odkázáni na obecné grafické editory s následujícími omezeními:

Nástroj Omezení pro cylindrické projekce
Adobe Photoshop / Illustrator Žádná podpora cylindrické projekce, žádný 3D náhled, žádná safe zone
Figma Plochý editor, plugin ekosystém omezený, žádná podpora výpočtu fyzikálních parametrů cylindru
Blender / Cinema 4D 3D nástroje vhodné pro finální rendering, ne pro kompozici a vrstvení 2D obsahu
360° video editory Zaměřené na sférické projekce (equirectangular), nikoliv cylindrické
Klíčový insight
ARena vyplňuje mezeru na trhu jako první webový editor specializovaný na cylindrické projekce s fyzikálně korektním výpočtem bezpečné zóny a real-time 3D náhledem.

Kapitola 3

Datový model

Celý datový model projektu je definován v souboru src/types/project.ts jako TypeScript typy. Tato centrální definice zajišťuje typovou konzistenci napříč celou aplikací.

3.1 Struktura projektu

Nejvyšší entitou je objekt Project, který obaluje veškerý obsah jednoho editačního projektu: 

type Project = {
  id:           string;        // UUID projektu
  name:         string;        // Název projektu
  version:      string;        // Verze schématu ("1.0.0")
  template:     Template;      // Parametry cylindrické scény
  startSceneId: string;        // ID výchozí scény (pro preview)
  assets:       Asset[];       // Knihovna médií (obrázky, videa, SVG)
  scenes:       Scene[];       // Pole scén projektu
  createdAt:    string;        // ISO timestamp vytvoření
  updatedAt:    string;        // ISO timestamp poslední změny
};

Šablona cylindru (Template)

Objekt Template definuje fyzikální parametry projekční plochy. Výchozí hodnoty odpovídají typickému cylindrickému sálu: 

type Template = {
  width:              number;  // Šířka canvasu v px (= obvod, defaultně 7741 px)
  height:             number;  // Výška canvasu v px (defaultně 2450 px)
  segments:           number;  // Počet cylindrických segmentů (defaultně 6)
  mode:               'cylinder-360';
  projectionHeightCm?: number; // Fyzická výška projekce v cm (default 250 cm)
  viewerHeightCm?:    number;  // Výška pozorovatele v cm (default 170 cm)
  safeRadiusCm?:      number;  // Poloměr bezpečné zóny v cm (default 60 cm)
};
Poznámka k rozměrům
Výchozí šířka canvasu 7741 px reprezentuje celých 360° obvodu cylindru. Jeden pixel tedy odpovídá přibližně 0,0465° (360° / 7741 px). Výchozí poměr stran 7741 : 2450 ≈ 3,16 : 1 odpovídá typickému cylindrickému sálu o výšce 250 cm a průměru přibližně 15 m.

3.2 Scény a objekty

Projekt obsahuje pole scén. Každá scéna je nezávislá kompozice objektů na cylindrické ploše: 

type Scene = {
  id:         string;         // UUID scény
  name:       string;         // Název scény
  notes?:     string;         // Volné poznámky designéra
  background?: string;        // Barva pozadí (hex)
  thumbnail?: string;         // Base64 miniatura (generovaná)
  objects:    SceneObject[];  // Pole objektů scény
};

type SceneObject = {
  id:         string;
  type:       SceneObjectType;  // 'image'|'svg'|'video'|'text'|'hotspot'|'shape'
  name:       string;
  x:          number;           // Levý okraj v px (souřadnice canvasu)
  y:          number;           // Horní okraj v px (souřadnice canvasu)
  width:      number;           // Šířka v px
  height:     number;           // Výška v px
  rotation:   number;           // Rotace ve stupních
  opacity:    number;           // Průhlednost 0.0–1.0
  visible:    boolean;
  locked:     boolean;
  zIndex:     number;           // Pořadí vrstev (vyšší = navrch)
  props:      SceneObjectProps; // Typ-specifické vlastnosti
  interactions?: Interaction[];
};

3.4 Interakce

Každý objekt může mít přiřazeno pole Interaction, které definuje chování v preview/přehrávacím režimu. Interakce mají strukturu trigger → action → target: 

type Interaction = {
  id:       string;
  trigger:  'onClick' | 'onSceneEnter' | 'onTimer' | 'onHover';
  action:   'goToScene' | 'showObject' | 'hideObject' | 'toggleObject'
           | 'setText' | 'playVideo' | 'pauseVideo';
  targetId?: string;   // ID cílového objektu nebo scény
  value?:    string;   // Volná hodnota (text, čas apod.)
  delayMs?:  number;   // Zpoždění akce v ms
};

Kapitola 4

Implementace klíčových subsystémů

4.1 2D editor — Konva

Jádrem editoru je komponenta Canvas2D využívající knihovnu Konva.js přes React wrapper react-konva. Konva poskytuje abstrakci nad HTML5 Canvas API s podporou:

  • Hierarchického stromu uzlů (Stage → Layer → Group → Shape)
  • Vestavěného hit-testingu pro výběr objektů
  • Transformer node pro interaktivní resize a rotaci výběru
  • Drag&drop se snap-to-guide logikou

4.2 Souřadnicový systém a zoom/pan transformace

Editor udržuje dvě sady souřadnic — screenové (pixely viewportu) a světové (pixely canvasu). Transformace mezi nimi závisí na aktuálním zoomu a posunu:

worldX = (screenX − panX) ÷ zoom // screen → canvas
screenX = worldX × zoom + panX // canvas → screen

Zoom k bodu (cursor-centric zoom)

Při zoom gestě (Ctrl + kolečko myši) musí canvas zůstat "přichycen" pod kurzorem. Toho se dosahuje dvoufázovou transformací:

// 1. Zachovej světovou polohu bodu pod kurzorem
worldX = (pointer.x − panX) ÷ zoom
// 2. Vypočítej nový pan tak, aby worldX zůstalo pod pointer.x
newPanX = pointer.x − worldX × newZoom

Zoom je omezen na rozsah [MIN_ZOOM = 0.02, MAX_ZOOM = 4.0]. Faktor zoomu na jeden krok kolečka je 1.12 (12% per krok).

Auto-fit při startu

Při prvním zobrazení canvasu se automaticky nastaví zoom tak, aby se celá projekční plocha vešla do viewportu s paddingem 32 px:

fitZoom = min( (viewport.w − 64) ÷ template.width, (viewport.h − 64) ÷ template.height )
panX = (viewport.w − template.width × fitZoom) ÷ 2 // vycentruj horizontálně

4.3 Matematický model — výpočet bezpečné zóny

Bezpečná zóna (safe zone) je horizontální pás na canvasu, který je v cylindrickém prostoru viditelný pro pozorovatele dané výšky. Výpočet vychází z fyzikálních parametrů cylindru uložených v Template.

Fyzikální model

Cylindr stojí svisle; pozorovatel stojí uvnitř na podlaze. Souřadnice y = 0 na canvasu odpovídá vrcholu cylindru (stropu), y = template.height odpovídá podlaze.

Výpočetní vzorce

Implementace funkce computeSafeZone(template) z src/types/project.ts:

pxPerCm = template.height ÷ projectionHeightCm // převod cm → px
eyeFromTopCm = projectionHeightCm − viewerHeightCm // vzdálenost očí od vrcholu cylindru
eyeLevelPx = eyeFromTopCm × pxPerCm // y-pozice úrovně očí na canvasu
topPx = max(0, eyeLevelPx − safeRadiusCm × pxPerCm) // horní hranice bezp. zóny
bottomPx = max(0, template.height − (eyeLevelPx + safeRadiusCm × pxPerCm)) // vzdálenost dolní hranice od dna

4.4 3D cylindrický náhled — React Three Fiber

3D náhled je implementován v komponentě CanvasShell3D pomocí React Three Fiber (deklarativní React wrapper nad Three.js / WebGL).

Geometrie cylindru

Geometrie je generována pomocí Three.js CylinderGeometry s parametry odvozenými z Template: 

// Vnější poloměr cylindru v 3D jednotkách
const outerRadius = 2.5;

// Výška cylindru zachovává poměr stran projekční plochy
const height = (template.height / template.width) * (Math.PI * 2 * outerRadius);

// Geometrie — 96 segmentů pro plynulý povrch
new THREE.CylinderGeometry(outerRadius, outerRadius, height, 96, 1, true)
// true = openEnded (bez víček)

Cylindr se renderuje jako dvě meshes:

  • Vnější povrch (FrontSide, opacity 0.18) — umožňuje spatřit cylindr zvenku
  • Vnitřní povrch (BackSide, opacity 1.0) — plná textura pro pohled zevnitř

Kamera a ovládání

// Počáteční pozice kamery — pohled zvenku do cylindru
camera={{ position: [0, 1.4, 6.5], fov: 40 }}

// OrbitControls omezení
enablePan={false}
minDistance={0.5}   // max zoom dovnitř
maxDistance={14}    // max zoom ven
maxPolarAngle={Math.PI / 1.6}  // omezení úhlu (ne zcela dolů)

Funkce renderSceneToCanvas() v src/lib/renderScene.ts renderuje scénu do pomocného HTML <canvas> elementu ve dvou průchodech:

  1. Průchod 1: Pozadí + průvodní linky (segmenty, mřížka)
  2. Průchod 2: Objekty scény seřazené dle zIndex (painter's algorithm)

Textura se renderuje v rozlišení 2048 px šíře (poměr stran zachován), nezávisle na aktuálním zoomu 2D editoru.

Historie (undo/redo)

Undo/redo je implementováno jako snapshot-based history: před každou destruktivní operací se volá pushHistory(), která uloží aktuální Project objekt do zásobníku history.past. 

// Struktura historického zásobníku
history: {
  past:   Project[],  // zásobník zpět (max 50 položek)
  future: Project[],  // zásobník vpřed (prázdní po nové akci)
}

Celý objekt Project se serializuje do historického zásobníku — stav je tedy plně obnoven včetně všech scén, objektů a jejich vlastností. Hloubka zásobníku je omezena na 50 kroků z důvodu paměťové efektivity.

4.7 Perzistence dat a autosave

ARena pracuje jako local-first aplikace — veškerá data jsou uložena v prohlížeči, bez nutnosti síťového připojení.

MechanismusPoužitíKapacita
localStorage Metadata projektů, seznam projektů ~5 MB (limit prohlížeče)
IndexedDB Binární obsah assetů (obrázky, videa jako base64/blob URL) Stovky MB (závisí na prohlížeči)

Autosave je implementován hookem useAutosave(), který reaguje na změny stavu Zustand store a ukládá projekt po 800ms debounce od poslední změny.

5.2 Popis komponent

TopBar

Horní navigační lišta obsahuje tři logické sekce oddělené flexbox layoutem:

  • Vlevo: logo (odkaz na ProjectPicker), editovatelný název projektu
  • Střed: přepínač 2D/3D, přepínače mřížky/segmentů/safe zone (icon buttons)
  • Vpravo: badge stavu uložení, tlačítko nápovědy, tlačítko Preview (gradient), ExportMenu

SidebarRail

Levá část se skládá ze dvou vrstev:

  • Nav rail (48 px) — ikony pro přepínání panelů + tlačítko hide/show všech panelů
  • Panel (260 px) — buď LayersPanel (strom objektů scény s drag-to-reorder) nebo AssetLibrary (miniatura assetů)

Toolbar

Plovoucí lišta nástrojů umístěná nad canvasem (absolutně pozicovaná, centrovaná horizontálně). Každý nástroj je draggable — přetažením na canvas se objekt vytvoří na konkrétní pozici. Kliknutím se objekt přidá do středu canvasu.

NástrojKlávesaTyp objektu
Text (T)—text
Hotspot—hotspot
Obdélník—shape (rect)
Elipsa—shape (ellipse)

Inspector

Pravý panel kontextově zobrazuje vlastnosti právě vybraného objektu (nebo vlastnosti scény, není-li nic vybráno). Sekce jsou: Editor (pozice, rotace), Layout (rozměry, vrstva), Appearance (průhlednost), typ-specifické sekce (Text, Fill, Hotspot…) a Action (správa interakcí).

SceneStrip

Spodní lišta zobrazuje miniatury všech scén projektu. Aktivní scéna je zvýrazněna accent barvou s pink glowem. Kliknutím na miniaturu se přepne aktivní scéna. Kontextové hover akce umožňují duplicitu nebo smazání scény.

Kapitola 7

Závěr a možná rozšíření

7.1 Shrnutí

ARena představuje funkční proof-of-concept specializovaného editorského nástroje pro cylindrické projekční scény. Aplikace úspěšně kombinuje:

  • Plnohodnotný 2D canvas editor s Konva.js (drag&drop, resize, rotace, vrstvení, snap k průvodcím)
  • Fyzikálně korektní výpočet bezpečné zóny pro pozorovatele různých výšek
  • Real-time 3D cylindrický náhled s live texturou z 2D editoru
  • Interaktivní systém hotspotů pro navigaci mezi scénami
  • Local-first architekturu s autosave a multi-formátovým exportem

7.2 Technická omezení

Aktuální omezení
  • Kapacita localStorage limituje velikost projektů s mnoha médii
  • Video objekty nejsou zahrnuty do 3D textury (HTML video nelze přímo použít jako WebGL textura bez dalšího zpracování)
  • Export PNG renderuje vždy v 2048 px šíři — pro very large format produkci by bylo žádoucí vyšší rozlišení
  • Žádná podpora kooperativní editace (single-user)
